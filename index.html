What is Claude Code?

Claude Code is Anthropic's official CLI tool that brings Claude directly into your terminal. It understands your codebase, edits files, runs commands, and automates your development workflow — all through natural language.

🖥️

Terminal-Native

Works where you work. No context switching between IDE and AI — Claude lives right in your shell.

📁

Codebase Aware

Automatically indexes your project. Claude understands your file structure, git history, and code patterns.

🔧

Direct Editing

Claude creates and edits files directly. No copy-paste — it reads your codebase and writes changes inline.

⚡

Command Execution

Runs shell commands, installs packages, runs tests, and manages git — all with your approval.

/init — Project Setup Made Easy

/init is the first command you should run in any project. It scans your codebase and generates a CLAUDE.md file that teaches Claude about your tech stack, coding conventions, build commands, and project structure.

Claude Code
/init

After running /init, review the generated CLAUDE.md — add domain-specific context, team conventions, and any gotchas. The better your CLAUDE.md, the smarter Claude becomes in your project.

/compact — Never Run Out of Context

Claude Code has a context window limit. When your conversation gets long (many back-and-forth messages, large file reads), you'll approach that limit. /compact intelligently summarizes the conversation so far, preserving key decisions, ongoing tasks, and important context — then frees up space for more work.

Compact with a hint

Add a focus hint so the summary keeps what matters: /compact keep the auth refactor context

Compact proactively

Don't wait until you hit the limit. Compact before starting a new major task — it's faster and preserves better context.

Check usage first

Run /context or /usage to see how full your context is. Compact when you're above 70%.

CLAUDE.md — Your AI Onboarding Doc

CLAUDE.md is loaded at the start of every session. Think of it as the onboarding document for your AI pair programmer. Claude reads it to understand how your project works and how you like to code.

Configuration Hierarchy (later overrides earlier)

  1. Enterprise — /etc/claude-code/CLAUDE.md
  2. User Global — ~/.claude/CLAUDE.md
  3. User Rules — ~/.claude/rules/*.md
  4. Project — ./CLAUDE.md or ./.claude/CLAUDE.md
  5. Project Rules — ./.claude/rules/*.md (path-scoped)
  6. Local Overrides — ./CLAUDE.local.md (auto-gitignored)

Keep CLAUDE.md under ~200 lines. Use @path/to/file imports for detailed reference docs. Move large instruction sets to .claude/rules/ or custom skills.

Permission System — Stay in Control

Claude Code's permission system lets you control exactly which actions Claude can take and when it needs your approval. Configure it with /permissions or /config → Permissions tab.

📖

Read-Only Tools

Read, Glob, Grep — safe to allow without prompt. Claude can explore your codebase freely.

✏️

Edit & Write

Edit, Write — may require confirmation. You review changes before they land.

⚡

Shell Execution

Bash — can be allowed with a denylist of dangerous commands, or always prompt.

Permission flow: Enterprise policy → User settings → Project settings → Session overrides. Start restrictive and loosen as you build trust.

Hooks — Enforce Your Rules Automatically

Hooks are event-driven scripts that fire at specific moments — before a tool runs, after a file is edited, when a session starts. Unlike CLAUDE.md (advisory), hooks are enforcement.

.claude/settings.json
{
  "hooks": {
    "PostToolUse": [{
      "matcher": "Edit|Write",
      "command": "npx eslint --fix \"$CLAUDE_FILE_PATH\""
    }]
  }
}

Common hook use cases: auto-lint after edits, run tests after file changes, block edits to .env files, enforce commit message formats. Configure in settings.json under "hooks".

Skills — Reusable Workflows On Demand

Skills are reusable instruction packs stored in .claude/skills/. Unlike CLAUDE.md (loaded every session), skills load only when invoked — keeping your context lean. Create a SKILL.md file to define a skill.

.claude/skills/deploy/SKILL.md
---
name: deploy
description: Deploy the application. Trigger: "deploy", "ship it"
---

# Deploy Workflow
1. Run tests: `npm test`
2. Build: `npm run build`
3. Deploy: `npm run deploy`
4. Verify: curl the health endpoint

Invoke skills explicitly with /deploy or let Claude auto-match them by trigger words in your message. Great for: deployment workflows, database migrations, onboarding new team members, release checklists.

Tips & Best Practices

💡

Be Specific

Instead of "fix the bug", say "fix the null pointer in the login handler when email is blank". The more context, the better the result.

🧠

Use CLAUDE.md

Create a CLAUDE.md file in your project root to teach Claude about your conventions, architecture, and preferences. It's like an onboarding doc for your AI pair programmer.

🎯

One Task at a Time

Break large tasks into smaller, focused requests. Claude performs best with clear, scoped objectives rather than vague mega-tasks.

✅

Review Before Committing

Always review Claude's changes before committing. Use git diff to inspect edits — Claude is powerful but not infallible.

📸

Provide Screenshots

For UI work, Claude can read screenshots. Share a screenshot of your current UI and describe what you want changed — it understands visual context.

🔒

Manage Permissions

Configure settings.json to control which actions Claude can take automatically. Start restrictive and loosen as you build trust.

Frequently Asked Questions

Claude Code is included with Claude Pro, Max, and Team subscriptions. API usage is billed separately through the Anthropic API at standard per-token rates.

Claude Code supports virtually all programming languages — Python, JavaScript, TypeScript, Go, Rust, Java, C/C++, Ruby, PHP, Swift, Kotlin, and many more. It also handles configuration formats like YAML, TOML, JSON, and Dockerfiles.

Claude Code processes your code through Anthropic's API to generate responses. By default, Anthropic does not train on your data. Enterprise customers can set up private processing through AWS Bedrock or GCP Vertex AI.

Claude Code requires an internet connection to communicate with Anthropic's API. There is currently no fully offline mode, though features like codebase indexing work locally.

Copilot is primarily an inline autocomplete tool that lives in your IDE. Claude Code is a full conversational agent in your terminal — it can read your entire codebase, execute commands, edit multiple files, manage git, and handle complex multi-step tasks autonomously.
